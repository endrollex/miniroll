<span style="font-size: 16pt; color: green;"><!--mi3-->Introduction:</span><!--mi4--><br />
<br />
minicode is a lightweight markup language for <a class="m4b" target="_blank" href="https://github.com/endrollex/miniroll"><!--m4a-->miniroll</a><!--m4--> easy editing, learn from <a class="m4b" target="_blank" href="https://www.phpbb.com/community/faq.php?mode=bbcode"><!--m4a-->BBCode</a><!--m4-->.<br />
The tags are indicated by square brackets [ ] surrounding a keyword.<br />
It is designed for Blog, thus let through unsafe HTML like &lt;script&gt;.<br />
minicode allow user to write HTML freely and unlimitedly.<br />
<br />
<span style="font-size: 16pt; color: green;"><!--mi3-->Syntax:</span><!--mi4--><br />
<!--m5--><ol><li>The minicode cannot nest:
<pre style="background-color: #D3D3D3;">[url]http://goo.gl/[/url] OK
[b][url]http://goo.gl/[/url][/b] This expression is ERROR!!!
</pre>Because nest will be complex.
In this situation, you can use HTML directly or use [lead_define].<br/><br/></li>
<li>Difference between minicode and BBCode:
<pre style="background-color: #D3D3D3;">[url]http://php.net/docs.php[PHP Manual[/url]
[color]#B22222]FireBrick[/color]
</pre>Note the tag option is close to the [tag] with a sigle square bracket:<br/>
[tag]option]...[/tag], or [tag]...[option[/tag]<br/>
Position of tag option depends on the implemented HTML sentence.<br/>
Do not add newline in the tag option of a minicode, it will be failed.

<br />
Mechanism of minicode is base on &quot;Search and Replace&quot;, the process is reversible and very simply.<br />
It abbreviated commonly-used HTML tags to short keywords, the article can mix HTML.<br />
The implemented HTML may contain a little comment markup &lt;!-- --&gt;, intend for reversion.<br />
Some keywords of minicode depends CSS, the <a class="m4b" target="_blank" href="https://github.com/endrollex/miniroll/blob/master/htmindex.css"><!--m4a-->default CSS</a><!--m4--> provided to show the predefined style.<br />
For this page process: <a class="m4b" target="_blank" href="http://endrollex.com/font/minicode_manual_m.txt"><!--m4a-->minicode</a><!--m4--> -&gt; <a class="m4b" target="_blank" href="http://endrollex.com/font/minicode_manual.txt"><!--m4a-->HTML article</a><!--m4--> -&gt; this page (Encapsulated)<br />
<br />
<span style="font-size: 16pt; color: green;"><!--mi3-->Construct:</span><!--mi4--><br />
<br />
minicode of miniroll is writed by PHP, HTML version is HTML5.<br />
You can modify all rules from <a class="m4b" target="_blank" href="https://github.com/endrollex/miniroll/blob/master/manage_dir/post.php"><!--m4a-->post.php</a><!--m4-->.<br />
minicode function is not independent from miniroll yet, but I think isolating the function code is easy.
